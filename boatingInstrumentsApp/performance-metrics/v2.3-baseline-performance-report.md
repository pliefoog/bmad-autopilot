3. **usePerformanceMonitor Hook** (`src/hooks/usePerformanceMonitor.ts`)
   - Component-level performance tracking
   - Render time analysis

## Baseline Targets (v2.3)

### UI Performance
- **Target FPS**: 60fps (smooth marine UI)
- **Max Render Time**: 16ms per frame
- **Memory Baseline**: <100MB initial load
- **Memory Ceiling**: <250MB with full dashboard

### Marine-Specific Requirements  
- **NMEA Processing**: <50ms latency per message
- **Widget Rendering**: <16ms per widget update
- **Data Refresh**: 1-10Hz depending on data type
- **Cross-Platform**: Consistent performance iOS/Android/Web

### Network Performance
- **NMEA Bridge Connection**: <100ms response time
- **Autopilot Commands**: <500ms total roundtrip
- **mDNS Discovery**: <5 seconds initial scan

## Performance Thresholds

### Critical Violations (Must Fix)
- FPS < 30 (poor user experience)
- Memory usage > 300MB (device impact)
- Render time > 32ms (janky animation)
- Memory leaks > 10MB/hour growth

### Warning Thresholds  
- FPS < 50 (noticeable lag)
- Memory usage > 200MB (optimization needed)
- Render time > 20ms (performance impact)
