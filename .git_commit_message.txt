fix(sensor-config): use enriched threshold defaults for slider fallback values

Problem:
- Slider thumbs not positioned at schema defaults
- Both critical and warning fell back to min.value
- Caused thumbs to overlap at minimum position (0 for depth)
- Slider bar not rendering properly with identical values

Root Cause:
- Fallback logic: criticalValueWatched ?? min.value
- Fallback logic: warningValueWatched ?? min.value
- Both used same fallback (min) instead of schema defaults

Solution:
- Use enrichedThresholds.display.critical?.value as first fallback
- Use enrichedThresholds.display.warning?.value as first fallback
- Only fall back to min/max if enriched defaults missing
- Ensures slider initializes with proper schema defaults (2m/2.5m for depth)

Fallback Chain:
1. Form value (user-saved or edited)
2. Enriched schema default (from ThresholdPresentationService)
3. Range boundary (min for critical, max for warning)

Impact:
- Depth slider: Initializes at 6.6ft (critical) and 8.2ft (warning) defaults
- Battery slider: Initializes at correct voltage defaults per chemistry
- Slider bar renders full width with visible separation between thumbs
