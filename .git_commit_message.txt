fix(sensor-config): show correct field label and live value for single-metric sensors

Problem:
1. Metric label showing sensor name "Depth Sounder" instead of field name "Depth"
2. Current value not displayed (always undefined)

Root Cause:
1. metricLabel logic:
   - Single-metric path returned sensorConfig.displayName (wrong)
   - Should return field label from first alarm field

2. currentMetricValue logic:
   - Used useMemo (computed once, never updates)
   - Needs reactive store selector for live updates

Solution:
1. Fixed metricLabel computation:
   - Multi-metric: Use selected metric's field label (unchanged)
   - Single-metric: Use first alarm field's label (NEW)
   - Example: depth sensor → "Depth" not "Depth Sounder"

2. Changed currentMetricValue to useNmeaStore selector:
   - Reactive to sensor data changes
   - Re-renders when value updates
   - Returns formattedValueWithUnit from MetricValue

Visual Result:
BEFORE:
┌─────────────────────────────┐
│ Depth Sounder    (no value) │
└─────────────────────────────┘

AFTER:
┌─────────────────────────────┐
│ Depth              8.2 ft   │
└─────────────────────────────┘

Live Updates:
- Value changes as sensor data arrives
- Real-time feedback for threshold setting
- Helps users set appropriate limits based on current conditions
