feat(sensor-config): show metric name and current value for single-metric sensors

Enhancement:
- Display metric name and current value for single-metric sensors (depth, wind, speed)
- Previously only multi-metric sensors (battery, engine) showed metric info
- Now single-metric sensors show equivalent static display

Implementation:
1. Added currentMetricValue to useSensorConfigForm computed values
   - Fetches MetricValue from sensor instance
   - Returns formattedValueWithUnit (e.g., "8.2 ft", "15.3 kt")
   - Uses same logic as multi-metric (metric key resolution)

2. Updated SensorConfigDialog metric display logic
   - Multi-metric: Shows dropdown selector (existing)
   - Single-metric: Shows static display with label + current value (NEW)
   - Conditional rendering based on requiresMetricSelection flag

3. Added metricDisplay styles
   - Horizontal layout with space-between
   - Label on left (metric name), value on right (current reading)
   - Primary color for value to highlight live data
   - Rounded background matching theme surface

User Experience:
- Depth sensor now shows: "Depth" → "8.2 ft"
- Wind sensor shows: "Speed" → "15.3 kt"
- Battery shows dropdown (unchanged, multi-metric)
- Live value updates with sensor data
- Consistent visual language across sensor types

Example:
┌─────────────────────────────────┐
│ Metric                          │
│ ┌─────────────────────────────┐ │
│ │ Depth            8.2 ft     │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
