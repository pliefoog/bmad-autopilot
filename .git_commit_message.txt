fix: ConfigFieldRenderer now renders sensor config input fields

Critical fix for ConfigFieldRenderer that was completely broken after
unified schema refactor. Input fields weren't rendering at all because
the component was trying to use old property names that don't exist.

Root Cause:
- ConfigFieldRenderer expected SensorFieldConfig type (doesn't exist)
- Component used field.uiType and field.valueType (don't exist in unified schema)
- Unified schema uses FieldDefinition with field.type property

Changes in ConfigFieldRenderer.tsx:
- Changed import from SensorFieldConfig → FieldDefinition
- Added FieldWithKey type (FieldDefinition + key property)
- Added uiType mapping: field.type → rendering type (textInput/numericInput/picker/toggle)
- Fixed textInput: field.valueType === 'string' → field.type === 'text'
- Fixed numericInput: field.valueType === 'number' → field.type === 'number'
- Fixed picker: field.valueType === 'string' → field.type === 'picker'
- Simplified picker options mapping (unified schema uses string[] not complex objects)
- Removed redundant field existence checks ('min' in field → just field.min)
